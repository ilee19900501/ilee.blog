核心概念

1.横切关注点：对哪些方法进行拦截，拦截后怎么处理
2.切面（aspect）:把原来糅杂在业务代码中的非业务代码抽取出来，把功能形同的放在一个类中形成一个切面（例如日志，权限校验等），
类是对物体特征的抽象，切面就是对横切关注点的抽象
3.连接点（joinPoint）:被拦截的方法（spring只支持方法类型的连接点）
4.切入点（pointCut）:对连接点进行拦截的定义
5.目标对象：代理的对象
6.织入（weave）:将切面应用到目标对象并导致代理对象的创建过程
7.advice 增强的方法（before,after）
8:一般切面Advisor,一个切面同时包含横切代码和连接点信息: Advisor=Advice+TruePointcut

cglib和jdk动态代理区别

首先看下spring源码里面是怎么决定使用哪个代理的

*spring中aop默认使用jdk动态代理，springboot2以后默认使用cglib来实现动态代理详解（可配置spring.aop.proxy-target-class=false使用jdk动态代理）
1.jdk动态代理只可以代理接口，因为最后的实现类要继承Proxy并实现该接口
public final class $Proxy0 extends Proxy implements Caculate {
2.cglib既可代理接口又可以代理实现类：
对接口进行代理的cglib，最后生成的源码是实现了该接口和Factory接口
对实现类进行代理的cglib，最后生成的源码是继承了实现类并实现了Factory接口

3.@Autowired UserServiceImpl userService;这里如果UserServiceImpl走了aop，使用jdk动态代理项目启东时则会报错
SpringBoot 正是出于这种考虑，于是在 2.x 版本中，将 AOP 默认实现改为了 CGLIB。
Spring Boot issue #5423:
Use @EnableTransactionManagement(proxyTargetClass = true) #5423
https://github.com/spring-projects/spring-boot/issues/5423

4.如果配置了使用jdk动态代理，代理类不是接口的话最终还是会走cglib.(例如controller层)
5.创建字节码层面，cglib会创建3个字节码文件，jdk会创建1个字节码文件
