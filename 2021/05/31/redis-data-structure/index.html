    redis单线程or多线程

    redis5.0（包括）版本之前都是单线程。

    • redis单线程：redis的单线程主要是指网络io和健值的读写操作，但是redis的其他功能，像持久化，异步删除，集群数据的同步都是由其他线程完成的。
      异步删除：redis有很多的数据类型尤其像是list , hash ,set , zset 时间复杂度为O(N)，当删除一个bigkey时需要很长时间，会阻塞redis其他操作，所以引入了非阻塞删除unlink命令。

    • redis单线程为什么这么快？
      因为redis的数据操作都是在内存中的，单线程减少了多线程之间的切换性能损耗问题。所以在操作redis的时候，尽量减少bigkey的操作，如keys*等。（如果redis做强一致性则会影响到性能，比如写一条命令就像持久化文件里同步一份）

    • redis单线程如何处理那么多的客户端并发？
      src/redis-benchmark get可以压测redis在1s之内能处理多少请求
      redis采用epoll实现多路复用（NIO）技术，将连接信息和事件放到队列中，依次放到文件事件分派器，事件分派器将事件分发给事件处理器。（在连接和读取数据的等待时间，不会阻塞在那，还可以处理其他请求）

    • redis6.0采用了多线程技术，主要对网络io做了优化

      io-threads 4 //设置多线程

      io-threads-do-reads no //设置io线程可读（没开启的话io线程负责write操作）


      所有的命令执行都是由主线程完成。
      单线程下执行一个命令需要：从socket连接read数据 parse成resp协议的数据 执行命令execute command 通过socket连接写入db write

      redis常用数据结构

    • string

    • hash
      hset key field value[field value]
      使用场景：购物车添加商品时候
      添加商品hset cart:1001 goodsId:100 1

         hset cart:1001 goodsId:101 1

      增加数量：HINCRBY cart:1001 goodsId:100 2
      获取购物车的所有商品hgetall cart:1001
      获取某个商品：hget cart:1001 goodsId:100
      商品总数：HLEN cart:1001
      删除某个商品：hdel cart:1001 goodsId:100
      优点：1.相比string占用内存和CPU小 2.相比string方便管理数据，多个数据在同一个key下管理 3.可以减少扩容rehash次数（string类型的比如一个用户多个字段，每个字段都set key->hash(key)->arr[]数组放不下时候会扩容进行rehash）
      缺点：1.容易产生bigkey 2.过期功能不能用在field上只能用在key上

    • list
      从左边添加元素：lpush color red blue green
      遍历：LRANGE color 0 10
      green blue red
      从左边取一个元素：lpop color
      实现栈数据结构：（FILO）lpush + lpop
      从右边添加元素：rpush color yelow
      从右边取一个元素： rpop color
      实现队列数据结构（FIFO）：lpush + rpop
      设置超时时间(如果取不到元素，则等待)；brpop key [] timeout
      实现阻塞队列：lpush + brpop
      说明：分布式环境下，要实现阻塞队列和栈数据结构，jdk提供的数据结构就用不了了，可以使用redis的
      使用场景:
      微博消息和微信公众号发消息(feed流)。
      lee关注了央视新闻和第一新闻，央视新闻推和第一新闻分别推送了1条消息，id为1008和1009。

      1. 用户量小的时候可以采用push（可以将用户分为活跃用户，非活跃用户）
        lpush msg:{lee的id} 1008
        lpush msg:{lee的id} 1009
        按时间先后顺序展示：lrange msg:{lee的id} 0 1
      2. 用户量大的时候可以采用pull（消息先发到队列里面去，每个用户去队列里面去拉取）

    • set
      sadd key value(不重复的元素)
      SMEMBERS key(查看所有元素)
      使用场景：

      1. 用户随机抽奖时
        sadd prize user01
        sadd prize user02
        sadd prize user03
        SRANDMEMBER prize 1 随机从里面抽取一个（不去除抽过的）
        spop prize 1 随机从里面抽取一个（去除抽过的）
      2. 朋友圈点赞
        用户user01，user02，user03都对like:01这条动态点赞
        sadd like:01 user01
        sadd like:01 user02
        sadd like:01 user03
        查看所有点赞：SMEMBERS like:01
        取消点赞：srem like:01 user01
        查看某个用户是否在点赞列表里面：SISMEMBER like:01 user01
        查看点赞用户数：SCARD like:01
      3. 关注模型（使用集合的交、并、差可实现）
        我关注的：my:watch-> {user01,user02,user03}
        张三关注的：zhangsan:watch-> {user07,user02,user09}
        李四关注的：lisi:watch-> {user03,user04,user09}
        1> 共同关注的人 ：我关注的好友列表和他的好友列表取交集 sinter my:watch zhangsan:watch
        2> 我关注的人也关注他：查看我关注的好友是不是存在他的好友列表里面
        查看我的好友user01是否张三也关注：SISMEMBER zhangsan:watch user01
        3> 我可能认识的人：思路：用他的好友列表和我的好友列表做查差集
        进入到李四的微博，我可能认识的人为 sdiff lisi:watch my:watch

      重点: 关注模型的作用：推荐的人越多，一个人关注的东西越多，则互联网搜取到你的数据越精确，以后给你推荐的数据则更精确

    • zset（有序集合）
      添加一个带分值的元素：zadd key score member[score member]
      分值从小到大排序：ZRANGE key start stop [WITHSCORES]
      分值从大到小排序：ZREVRANGE key start stop [WITHSCORES]
      使用场景：展示微博当前热搜榜前七消息
      news:01和news:02的消息每点击一次，分值加一
      ZINCRBY hotnews:20210606 1 news:01
      ZINCRBY hotnews:20210606 1 news:02
      展示前七个分值最大的消息：ZREVRANGE hotnews:20210606 0 6 withscores

    • 其他高级命令
      scan：渐进式遍历键
      SCAN cursor [MATCH pattern] [COUNT count]
      scan 参数提供了三个参数，第一个是 cursor 整数值(hash桶的索引值)，第二个是 key 的正则模式， 第三个是一次遍历的key的数量(参考值，底层遍历的数量不一定)，并不是符合条件的结果数量。第 一次遍历时，cursor 值为 0，然后将返回结果中第一个整数值作为下一次遍历的 cursor。一直遍历 到返回的 cursor 值为 0 时结束。
      注意：scan并不能保证完整的遍历出来所有的键， 这些是我们在开发时需要考虑的。

    • bitmap(位图)
      通过一个bit位来表示某个元素对应的值或者状态，其中的key就是对应元素本身，value对应0或1，我们知道8个bit可以组成一个Byte，所以bitmap本身会极大的节省储存空间。
      使用场景：千万级用户登录统计
      设置offset的位置为1 SETBIT key offset value
      例子：统计6月12号这天登录用户数量
      用户id100的用户登录可设置offset100的位置为1：SETBIT login_user_0612 100 1
      统计1的个数：BITCOUNT login_user_0612
      例子：统计6月12号和13号连续登录用户数量
      BITOP and login_user_0612-0613 login_user_0612 lohin_user_0613
      BITCOUNT login_user_0612-0613

      BITOP AND destkey key [key …] ，对一个或多个 key 求逻辑并，并将结果保存到 destkey 。
      　BITOP OR destkey key [key …] ，对一个或多个 key 求逻辑或，并将结果保存到 destkey 。
      　BITOP XOR destkey key [key …] ，对一个或多个 key 求逻辑异或，并将结果保存到 destkey 。
      　BITOP NOT destkey key ，对给定 key 求逻辑非，并将结果保存到 destkey 。
